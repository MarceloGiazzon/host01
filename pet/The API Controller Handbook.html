The API Controller Handbook

A complete guide for non-developers to manage and generate data.

1. Welcome & Purpose

Welcome to the API CRUD Controller v3. Even if you have never heard of an "API" before, this tool is designed to help you interact with a database easily.

Think of this application as a Universal Remote Control for a digital warehouse. Instead of writing complex computer code to see what is in the warehouse or to add new items, you use this simple interface with buttons and forms.

The Goal: To make it easy to view existing information, fix mistakes, and create large amounts of test data (like a "Factory Mode") without needing a programmer.

2. Basic Concepts (No Tech-Speak)

To use this tool effectively, you only need to understand four basic actions, often called CRUD:

C - Create (POST)

Adding a brand new record to the system. Like filling out a registration form for the first time.

R - Read (GET)

Looking up an existing record. Like searching for a book in a library by its ID number.

U - Update (PUT)

Changing information on a record that already exists. Like updating your phone number on a profile.

D - Delete

Removing a record permanently from the system. (Use with caution!)

4. Special Safety Features

The "Wait & Check" Modal

Whenever you try to Create or Update something, the system doesn't send it immediately. A window pops up showing you exactly what the computer is about to "say" to the server. This is your chance to catch typos!

Traffic Logs

At the bottom of the screen, there is a black box that scrolls text. This is like a Receipt Printer. It shows you exactly when a request was sent and if the server said "Okay" (Green) or "Error" (Red).

5. The "Magic" of Batching

The most powerful part of this tool is the Batch Generation. Imagine you need 50 users to test your new website. Instead of typing them one by one, the Batch tabs do this:

  • Creative Names: It picks from a list of real-sounding names and cities.
  • Smart Stories: For Activities, it writes creative sentences like "A routine search for the Classic Book Edition" instead of just "User did activity".
  • Relationship Logic: It automatically links the new activities to users and products that already exist in your database, so the data "makes sense."

Step-by-Step Navigation Guide

Tab 1: Activities / Products / Users (The Editor)

How to use: Use these to look up one specific thing. Type the ID number in the first box and click FIND. The data will fill the form automatically. You can then change a name or description and click CREATE to save it.

Tab 2: Batch User / Batch Product (The Factory)

How to use: Enter a number in the "Quantity" box (e.g., 10) and click GENERATE. You will see 10 colorful cards appear with names and pictures. If they look good, click CREATE ALL to upload them all at once.

Tab 3: Batch Activity (The Storyteller)

How to use: Click SYNC & GENERATE. The tool will first "talk" to your database to see which Users and Products are available. Then, it will create realistic scenarios linking them together. Click CREATE ALL to save these interactions.

The Bottom Panels (The Monitor)

How to use: Look at the Network Traffic (left) to see the history of your session. Look at the Last Response (right) to see the technical details of the very last thing you clicked. If you see a "200" or "201", you succeeded!
